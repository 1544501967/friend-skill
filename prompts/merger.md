# 增量合并逻辑

## 任务

将新收集到的原材料与现有 Friendship Memory 和 Persona 进行合并。

## 合并原则

1. **追加不覆盖**：新信息追加到现有内容，不替换已有内容
2. **冲突标记**：新旧信息矛盾时，用 `[conflict]` 标记，等待用户确认
3. **时间线插入**：新事件按时间顺序插入到时间线中
4. **证据升级**：如果新信息证实或否定了原有推测，更新置信度

## 合并流程

### Memory 合并

1. 读取现有 `memory.md`
2. 分析新原材料，提取新增的记忆点
3. 按以下优先级合并：
   - **时间线**：新事件按时间插入
   - **共同记忆**：新记忆追加到对应分类
   - **日常模式**：新数据可以更新统计信息
   - **印象深刻的事**：如用户提供新的，追加
   - **Inside Jokes**：新增的追加
4. 标注合并来源：`[来自：{source_file_or_date}]`

### Persona 合并

1. 读取现有 `persona.md`
2. 分析新原材料，提取新增的性格/行为信息
3. 按以下优先级合并：
   - **说话风格**：新增的口头禅、语气词追加
   - **社交模式**：新的行为观察补充
   - **情绪表达**：新的表达方式补充
   - **处事方式**：新的决策模式补充
4. 如果新信息与已有信息矛盾：
   - 标记 `[conflict: 原有={old} vs 新={new}]`
   - 询问用户以哪个为准

## 冲突处理

```
发现信息冲突：

  已有信息：{old_info}
  新信息：{new_info}

  以哪个为准？
  [1] 保留原有
  [2] 使用新的
  [3] 两个都保留
```

## 写入

合并完成后：
1. 用 `Edit` 工具更新 `memory.md`
2. 用 `Edit` 工具更新 `persona.md`
3. 备份当前版本
4. 重新生成 `SKILL.md`
5. 更新 `meta.json` 的 version 和 updated_at
