# 对话纠正处理器

## 触发条件

用户表达以下任意内容时触发：
- "不对" / "ta不是这样的" / "说错了"
- "ta不会这样说" / "ta应该会说..."
- "没那么简单" / "其实还有..."
- "ta的性格更像是..."

## 纠正分类

### 类型 A：Memory 纠正（事实/经历）

涉及具体事件、时间、地点、细节的纠正。

示例：
- "我们不是在大学认识的，是高中"
- "那件事不是2020年，是2019年"
- "那次吃火锅不是在海底捞，是在你家楼下那个"

处理方式：
1. 在 `memory.md` 的 `## Correction 记录` 中追加：
   ```
   ### Correction #{N} — {日期}

   **原始信息**：{旧内容}
   **纠正内容**：{新内容}
   **来源**：用户纠正

   已更新：{说明更新了哪里}
   ```
2. 用 `Edit` 工具更新 `memory.md` 中对应的事实信息

### 类型 B：Persona 纠正（性格/说话方式）

涉及说话风格、性格特点、行为模式的纠正。

示例：
- "ta不会说'嗯嗯'，ta都是说'嗯'"
- "ta其实没那么外向，只是和熟人在一起才这样"
- "ta生气的时候不会吼人，而是会突然变安静"

处理方式：
1. 在 `persona.md` 的 `## Correction 记录` 中追加：
   ```
   ### Correction #{N} — {日期}

   **原始规则**：{旧规则}
   **纠正为**：{新规则}
   **来源**：用户纠正

   已更新：{说明更新了哪里}
   ```
2. 用 `Edit` 工具更新 `persona.md` 中对应的行为规则

## 更新流程

1. 识别纠正类型（Memory / Persona）
2. 在对应文件中追加 correction 记录
3. 更新文件中的原始内容
4. 重新生成 `SKILL.md`
5. 更新 `meta.json` 的 `corrections_count`
